Quick Start

Start the controller, approve workers, set controller render defaults, register a project, create a render setup, review fast validation, then queue the render. Workers receive controller-owned job payloads and verify local project, Unreal, output, and log paths before launching Unreal.

  1. Confirm the worker appears online and approved.
  2. Set a project .uproject path reachable by that worker.
  3. Choose a render mode: saved MRQ queue, or single Level Sequence plus MRQ config preset.
  4. Check Command Preview and Validation before pressing Start render.
  5. Use Job Details for command, diagnostics, raw logs, and output artifacts.

Render Modes

Saved MRQ Queue uses a queue asset that already contains the render jobs. RenderFarm passes -MoviePipelineConfig only. Level Sequence is intentionally blank in this mode.

Single Level Sequence + MRQ Config Preset passes both -LevelSequence and -MoviePipelineConfig. Use this when the sequence is selected by RenderFarm and the MRQ asset is a primary config preset.

Unreal Path Rules

Filesystem paths point to files, such as D:\Shows\Demo\Demo.uproject. Unreal asset paths use mount points, such as /Game/Render/HighQuality.HighQuality. Do not include /Content; Unreal omits that part for asset references.

Object paths repeat the asset name after the dot. Map arguments are package paths, not object paths, so /Game/Maps/Main.Main becomes /Game/Maps/Main. Simple map names such as V2 and Minimal_Default1 are valid when your project/queue can resolve them.

Saved MRQ Queue Paths

Queue mode accepts package-style queue paths such as /Game/RenderConfig or /Game/Cinematics/myRenderQueue. RenderFarm does not append .AssetName in queue mode because Unreal's command-line queue examples commonly use the queue package path.

If you copied a .uasset path from disk, D:\Project\Content\RenderConfig.uasset is safely converted to /Game/RenderConfig for queue mode.

MRQ Config Preset Paths

Single Sequence mode uses a Movie Pipeline Primary Config object path, such as /Game/RenderConfig.RenderConfig. If you enter /Game/RenderConfig in this mode, RenderFarm may append .RenderConfig because this argument expects an object path.

Fast validation checks shape and quoting only. It will say a path format looks valid, not that the asset exists. Use deep Unreal validation when you need asset-existence proof.

Output Verification

RenderFarm accepts non-empty render artifacts with these extensions: .mov, .mp4, .avi, .mkv, .png, .jpg, .jpeg, .exr, .tif, .tiff, .bmp, .wav, .aif, and .aiff. PNG files are not required.

If Unreal reports a completed movie render but no artifacts are found in the expected folder, the job can complete with a warning so you can inspect whether the MRQ preset wrote elsewhere. Common fallback folders are Saved/MovieRenders, Saved/Screenshots, and Saved/VideoCaptures.

Network And Mapped Drives

Prefer UNC paths such as \\SERVER\RenderFarmOutput for shared output roots. Mapped drives can disappear when a worker runs as a service because services may run under a different Windows account. If output validation fails, verify the worker account can create and delete a small file in the output folder.

Common Errors and Fixes

PowerShell ParserError from a quoted executable

When manually testing in PowerShell, prepend & before a quoted executable path. The C# app does not include & because it launches the executable directly.

Failed to find Pipeline Configuration asset to render

The MRQ config or queue preset was not found. Queue mode can use /Game/RenderConfig; Single Sequence config mode usually needs /Game/RenderConfig.RenderConfig.

Render launches game but does not render

Check that the saved queue contains jobs, or that Single Sequence mode has a valid Level Sequence and MRQ config preset.

No PNG files found but MOV exported

This is valid. RenderFarm accepts movie files such as .mov and reports detected extensions in Job Details.

Output written to wrong folder

Inspect Job Details and the MRQ Output settings. If fallback scanning finds files, RenderFarm reports the actual folder.

Mapped drive not visible to worker process

Use a UNC path or run the worker under an account that owns the mapped drive.

RLPlugin is Incompatible

Disable, rebuild, or update the plugin for this Unreal version before command-line rendering.

Datasmith/C4D material dependency missing

Enable/install Datasmith Content or repair/reimport affected C4D materials.

Troubleshooting Checklist

  1. Confirm the worker is online, approved, and idle.
  2. Open Command Preview and confirm the map, queue/config, and sequence mode are correct.
  3. Verify the project .uproject and UnrealEditor-Cmd.exe exist on the worker machine.
  4. Verify the output folder is writable by the worker process account.
  5. Open Latest Log Diagnostics, then expand Raw Log if the summary is not enough.
  6. For asset-existence questions, run deep validation in Unreal rather than relying on fast path validation.

Validation Badges

Ready means fast validation found no blockers. Review means a safe normalization or warning exists. Blocked means the render cannot be submitted until the top blocker is fixed. Fast validation does not prove assets exist inside Unreal.
